## Documentation Philosophy

Our documentation follows these principles:

1. **Clear and Concise** - Get to the point quickly
2. **Well-Organized** - Easy to find information
3. **Consistent** - Same style and terminology throughout
4. **Maintainable** - Easy to keep up to date
5. **Accessible** - Appropriate for target audience

See [DOCUMENTATION_STANDARDS.md](DOCUMENTATION_STANDARDS.md) for detailed guidelines.

## Keeping Documentation Updated

When you change the code:
1. Update relevant documentation in the same PR
2. Check all affected cross-references
3. Follow the standards in [DOCUMENTATION_STANDARDS.md](DOCUMENTATION_STANDARDS.md)
4. Request documentation review
